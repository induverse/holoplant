# 3D Models

# Introduction

This guide will explain how to add 3D models to the project, and how to position them in the AR environment.

# 3D format

Holoplant supports 3D Models in the glTF format.

glTF is a file format for 3D scenes and models using the JSON standard. It is an API-neutral runtime asset delivery format developed by the Khronos Group 3D Formats Working Group. This format is intended to be an efficient, interoperable format with minimum file size and runtime processing by apps. As such, its creators have described it as the "JPEG of 3D." glTF also defines a common publishing format for 3D content tools and services.

To upload the files to Holoplant the glTF assets need to be combined in one .glb file. (GLB file format is a binary form of glTF that includes textures instead of referencing them as external images)

There are various 3rd party tools available to convert glTF in GLB. For example: https://products.aspose.app/3d/conversion/gltf-to-glb

There are also multiple 3rd party tools to convert other 3D formatis into glTF. For example: https://blackthread.io/gltf-converter/

# Polygon count limitations

Microsoft recommends to limit the amount of polygons to 100,000 This corresponds to about 150,000 vertices. Other AR hardware will have a similar limitation, or even less. It is possible to show bigger models, but the performance will start degrading.

A quick way to check the amount of vertices of a 3D model is to open it with Microsoft 3D Viewer (Included in Windows 10). Then on the top-right click on 'States & Shading'.

For example this turbine has 169.700 vertices, which is just over the limit. If you would display this model in the hololens, it does not allow any space for other models.

So practically you will only want to use models with a low poly count, less than 10K vertices, and preferably even lower than 1000 vertices. For example this robot arm only has 806 vertices.

Holoplant is not a design tool, but meant for large-scale plant operation, maintenance and training. So the models can and should be simple.

# Download an example file and prepare a preview image

For this guide we will work with the 3D model of a gear:

Download the file from this location: http://induverse.com/wp-content/uploads/2020/09/gearwheel0001.glb

Before uploading we should prepare a preview image, so users browsing the media library can see how the model looks without loading it. Create the preview image by opening the Model in Microsoft 3D Viewer, and taking a screenshot. (Use greenshot or your favourite screenshot tool) Save the screenshot uder .png format, for example gearwheel0001.png

# Upload the asset to the Holoplant Server

Navigate to Media Management > Media Collections and create new new Media collection. Name the collection 'Equipment'

After renaming the collection, add a new media object:

Then click the 'Edit' button:

Change the name of the media object to 'Gear'. Then drag and drop the .glb file in the 'Upload Media' field, and the preview image in the 'Upload Preview' field. Then click 'Upload':

This should result in both the image preview and model being shown:

When navigating back to the Equipment Collection the preview image should also be visible:

# Load an instance of this model in the AR environment

Once a model is available in the media library it can instantiated in the AR environment.

Open the hand menu and select 'Add Model':

Select the 'Equipment' collection:

Add the 'Gearwheel':

This should result in a model of the Gearwheel being loaded, with a blue bounding box around it:

# Moving, rotating and resizing the 3D model

The model can can moved by grabbing it with one hand. Grabbing it with two hands allows for changing both the position, rotation and scale simultaneausly.

More fine-grained control for the rotation and scale is also possible by grabbing the bounding box at the edges or the cornes with two fingers.

# Removing the model and manipulation from the engineering environment

For now the model can only be removed from the engineering environment.

Navigate to the ModelInstances under Environments > Default > OriginAnchor:

This will show a list with all 3D model instances in the environment. In our example there is only one instance:

From here the 3D Model Instance can be removed, but also repositioned, rotated and scaled.

# Roadmap

In the future it will be possible to:

  • [ ] programmatically interact with the models
  • [ ] show some models only during specific steps in workflow (for example to show where something needs to be installed on the machine)
  • [ ] lock the models in place for normal users
  • [ ] automatically hide models when the distance to the user is too high, allowing an unlimited amount of models to be positioned in a bigger environment

# Final Word

As mentioned above, the total sum of all the vertices of all models in the environment should be less than 150.000.

This limitation will be removed in the future when the auto-hiding of te models will be implemented.
